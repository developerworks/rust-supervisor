# ADR-011: 策略评估管线固定顺序 (budget → meltdown → backoff)

- **日期**: 2026-05-18
- **状态**: Accepted

## 背景

重启策略评估包含预算检查、熔断检查和退避计算. 执行顺序影响结果语义.

## 可选方案

- 方案 A: 按任意顺序评估, 合并所有条件后决定.
- 方案 B: 固定 order: budget → meltdown → backoff.

## 决策

选择方案 B.

## 理由

- 预算不足时直接拒绝, 不需要经过熔断和退避计算 (性能优化).
- 熔断触发后不计算退避延迟 (熔断优先级高于退避).
- 固定顺序消除不确定性, 行为可预测.
- 顺序在 `SupervisionPipeline` 的 6 个 stage 中明确实现.

## 后果

- 正面: 性能优化: 预算不足时跳过后续计算.
- 正面: 行为确定性: 相同输入总是相同输出.
- 正面: 测试可精确断言管线中间状态.
- 负面: 需要文档明确固定顺序, 避免使用者误解.
